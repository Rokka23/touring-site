# 🏍️ ツーリングスポットナビ

バイク・車でのツーリングにおすすめのスポットを、地図上で探して計画できるWebアプリです。

🔗 **公開URL**: (デプロイ後にここへ追記)

## 概要

全国のツーリングスポットをエリア・季節・難易度などの条件で絞り込み、Googleマップ上で確認しながら、行きたい場所を「プラン」や「お気に入り」に保存できるアプリです。

## 主な機能

- 🗺️ **地図表示**: Google Maps上にツーリングスポットをピン表示
- 🔍 **絞り込み検索**: エリア・タイプ(山/海など)・季節・難易度で条件検索
- 📝 **プランメモ**: 気になったスポットをプランに追加し、ツーリング計画を作成
- ❤️ **お気に入り登録**: スポットをお気に入りに保存していつでも見返せる
- 📍 **詳細ページ**: 各スポットの住所・見どころなどの詳細情報を表示
- 📱 **レスポンシブ対応**: PC・タブレット・スマホで表示を最適化

## 使用技術

| 分類 | 技術 |
|---|---|
| フロントエンド | React 19, React Router v7 |
| ビルドツール | Vite |
| 地図 | Google Maps JavaScript API (`@react-google-maps/api`) |
| スタイリング | CSS Modules |
| アイコン | react-icons |
| その他 | Swiper |

## ディレクトリ構成

各ディレクトリ・ファイルの役割は以下のようになっています。

```
src/
├── assets             # 画像などの静的ファイル
├── components         # 再利用可能なコンポーネント
├── context            # 状態管理
├── data               # アプリで使用するデータの管理
├── pages              # ページコンポーネント
├── utils              # 共通関数
├── App.jsx            # アプリケーションのルートコンポーネント
└── main.jsx           # エントリーポイント
```

## 工夫した点

- スポットのデータをエリア・季節・難易度などでフィルタリングするロジックを独自実装
- お気に入り・プラン情報をlocalStorageで永続化し、リロードしても状態が保持される設計
- CSS Modulesによりコンポーネント単位でスタイルを分離し、クラス名の衝突を防止
- 画面幅に応じて地図のズームレベルを動的に調整

## セットアップ方法

```bash
# リポジトリをクローン
git clone <このリポジトリのURL>
cd touring-react

# 依存パッケージをインストール
npm install

# 環境変数を設定
cp .env.example .env
# .env を開き、Google Maps APIキーを設定してください

# 開発サーバーを起動
npm run dev
```

## 環境変数

Google Maps を利用するため、APIキーの設定が必要です。

```
VITE_GOOGLE_MAPS_API_KEY=あなたのGoogle Maps APIキー
```

[Google Cloud Console](https://console.cloud.google.com/google/maps-apis/credentials) からAPIキーを取得してください。
